# Milestone 8 — Score on Enemy Defeat

A small, self-contained gameplay addition to the arena shooter: **killing
an enemy awards points**, and the on-screen score climbs as you clear the
arena. The demo win: on **Play**, move and shoot — every enemy you destroy
bumps the HUD score, and the score stops counting once you die.

This is a **pure-code change**. The scene, prefabs, HUD, and `GameManager`
already exist and are fully wired — do **not** rebuild, re-wire, or edit
any `.unity` / `.prefab` / `.asset` file.

## Context

The arena shooter is already playable and wired
(`Assets/Scenes/Main.unity`, build index 0):

- `GameManager` (in the `Gameplay` assembly) owns the score:
  `public int Score`, `public void AddScore(int amount)` — which is
  already a no-op once `IsGameOver` is true — and `TriggerGameOver()`.
- `HUDScoreDisplay` already reads `GameManager.Score` every frame and
  renders it in the Canvas HUD, so **any score change is visible on
  screen with zero UI work**.
- `Enemy` (`Assets/Prefabs/Enemy.prefab`, `Gameplay` assembly) takes
  damage from bullets and calls a private `Die()` when its health hits
  zero. It already resolves scene singletons at runtime via
  `FindFirstObjectByType<PlayerController>()` in `Awake` — use the **same
  pattern** to reach the `GameManager`. A prefab cannot hold a reference
  to a scene object, so resolve it at runtime; do **not** add an
  Inspector field that would need scene wiring.

Hard constraints: **C# only**, all changes inside the existing `Gameplay`
assembly. No new assemblies, no scene/prefab/asset edits. Build on the
existing `GameManager.AddScore` — do not add a second, parallel score.

## Features

### Feature: Enemies award score when destroyed

- [ ] `Enemy` has a serialized `scoreValue` field (`int`, default `10`)
      editable in the Inspector, with a public way for tests to set it
      (e.g. extend the existing `Configure` method).
- [ ] When an `Enemy` dies from damage (its health reaches zero), it adds
      `scoreValue` to the scene's `GameManager` exactly once via
      `GameManager.AddScore`. The `GameManager` is resolved at runtime
      (e.g. `FindFirstObjectByType<GameManager>()`), the same way `Enemy`
      already finds the `PlayerController`.
- [ ] Scoring happens **only** inside the existing `Die()` (death by
      damage), so an enemy removed by spawner cleanup awards nothing, and
      no points are added once the game is over (rely on `AddScore`'s
      existing game-over guard — don't duplicate it).
- [ ] A PlayMode test creates a `GameManager` and an `Enemy`, damages the
      enemy until it dies, and asserts `GameManager.Score` increased by
      the enemy's `scoreValue`; a second test asserts no score is added
      after `TriggerGameOver()` has fired.
