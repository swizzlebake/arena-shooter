# Milestone 9 — Polish: Art, VFX & Main Menu

Dresses the complete M8 round into a presentable prototype. Generated
sprite art replaces the placeholder squares, hit-flash and death-burst
VFX add juice, and a main-menu scene wraps the loop (Play → game, Quit).
After this milestone the prototype is a complete, good-looking top-down
arena survival shooter.

## Context

Builds on the full round shipped in M8 (`v1.0.0`): move/shoot/kill core
loop, waves + object pooling, health, score, restart, game-over, and the
runtime audio manager already exist (`PlayerController`, `Weapon`,
`Enemy`, `EnemySpawner`, `GameManager`, `ObjectPool`, `WavePlan`,
`AudioManager`, `HUDManager`, `GameOverPanel`, and `Main.unity`). This
milestone dresses that round — do not rewrite it.

Art is produced by the Artist/ComfyUI backend; VFX uses Unity
`ParticleSystem` + sprite tint (the shader-engineer may add a simple
additive/glow material but no custom shader is required). Menu flow uses
`SceneManager`. Keep any feedback maths (e.g. camera shake) as pure
`Game` logic so it stays testable. See `.swizz/context.md` for layout and
constraints. Audio is already wired via `AudioManager` — the new VFX
should fire alongside the existing hit/death/fire SFX, not replace them.

## Features

### Feature: Hit & death VFX (shader-engineer + Gameplay)

- [ ] `Assets/Scripts/Gameplay/HitFlash.cs`: on taking damage an enemy
      briefly flashes (color/material tint) then restores to its normal
      appearance — no permanent tint change.
- [ ] `Assets/Prefabs/DeathBurst.prefab` (a `ParticleSystem`) spawns at
      an enemy's position on death and auto-despawns; a muzzle-flash
      effect plays on fire.
- [ ] PlayMode tests: hitting an enemy sets then clears the flash within
      a short window; killing an enemy spawns a burst that cleans itself
      up.

**Notes:**
- Reuse the existing pooling seam (`ObjectPool`/`ObjectPoolManager`) for
  the death burst where practical, or self-destroy the prefab — either is
  acceptable.

### Feature: Screen feedback (optional polish, Game + Gameplay)

- [ ] `Assets/Scripts/Game/Shake.cs` (pure): given an intensity and time,
      returns a decaying camera offset; EditMode-tested to decay to zero.
- [ ] A `CameraShake` MonoBehaviour applies it to `Main Camera` on player
      hit / enemy death. Optional — cutting it does not fail the
      milestone.

## UI Features

### UI Feature: Sprite art pass (Artist)

- [ ] PNGs under `Assets/Sprites/` for a player ship/character, an enemy,
      a bullet, and a tiling background — a neon-on-dark arcade look that
      matches the dark camera background.
- [ ] Each imports as a 2D `Sprite` with consistent pixels-per-unit for
      the arena scale.
- [ ] Sprites are assigned to the right renderers: `Player` → player
      sprite, `Enemy` prefab → enemy sprite, `Bullet` prefab → bullet
      sprite; the background renders behind all gameplay and does not
      occlude it.
- [ ] No placeholder squares remain on the player, enemy, or bullet.

**Notes:**
- Aesthetic quality is best-effort — QA verifies the assets exist, import
  as Sprites, and are assigned, not artistry. A flat tinted primitive is
  an acceptable fallback if a generation fails or ComfyUI is unreachable.

### UI Feature: Main menu & flow (UI)

- [ ] `Assets/Scenes/MainMenu.unity` with a title and Play / Quit
      controls (uGUI buttons or key prompts). Play loads `Main.unity`;
      Quit exits play/app.
- [ ] Build Settings contain both scenes with `MainMenu` at index 0 and
      `Main` at index 1.
- [ ] `GameOverPanel` gains a "Main Menu" option that returns to
      `MainMenu` (alongside its existing Restart, which reloads the
      gameplay scene).
- [ ] A test asserts both scenes are in Build Settings in the correct
      order and that the Play control loads the gameplay scene.

## Architecture Notes

Art via the Artist/ComfyUI backend (`[agents.artist]` in `swizz.toml`,
ComfyUI workflow `.swizz/workflows/sd15-simple.json`). VFX via
`ParticleSystem` + `SpriteRenderer` tint; the death burst is a pooled or
self-destroying prefab. Menu transitions via `SceneManager.LoadScene`.
Camera shake stays pure-logic for testability. Scene/asset wiring
(sprite assignment, `MainMenu.unity`, Build Settings, prefab edits) is
the `unity-editor-engineer`'s job over MCP on top of the compiled C#.

## Out of Scope

Multiple enemy types / bosses, power-ups & upgrades, persistence /
high-score tables, and mobile/touch input — future milestones. Audio is
already delivered in M8 and is not re-done here.

## Dependencies

- **M7 + M8 complete** — the released core loop + game feel + audio round
  (they are; `v1.0.0`).
- Unity 6000.x with Input System + Physics2D + Test Framework; the
  IvanMurzak/Unity-MCP package installed and the Editor open.
- **ComfyUI reachable at `http://localhost:8188`** (the Artist runs this
  milestone) with `.swizz/workflows/sd15-simple.json` present. If ComfyUI
  is down, the art pass falls back to tinted primitives rather than
  blocking the round.
- An inference model available; the `unity-editor-engineer` /
  `shader-engineer` remain pinned to DeepSeek cloud (key exported per the
  node setup).
