# Milestone 3 — Polish: Art, VFX & Main Menu

Dresses the complete M2 round into a presentable prototype. Generated
sprite art replaces the placeholder squares, hit-flash and death-burst
VFX add juice, and a main-menu scene wraps the loop (Play → game, Quit).
After this milestone the prototype is a complete, good-looking top-down
arena survival shooter.

## Context

Builds on the full round from M2. Art is produced by the Artist/ComfyUI
backend; VFX uses Unity `ParticleSystem` + sprite tint (the
shader-engineer may add a simple additive/glow material but no custom
shader is required). Menu flow uses `SceneManager`. Keep any feedback
maths (e.g. camera shake) as pure `Game` logic so it stays testable. See
`.swizz/context.md` for layout and constraints.

## Features

### Feature: Hit & death VFX (shader-engineer + Gameplay)

- [ ] `Assets/Scripts/Gameplay/HitFlash.cs`: on taking damage an enemy
      briefly flashes (color/material tint) then restores to its normal
      appearance — no permanent tint change.
- [ ] `Assets/Prefabs/DeathBurst.prefab` (a `ParticleSystem`) spawns at
      an enemy's position on death and auto-despawns; a muzzle-flash
      effect plays on fire.
- [ ] PlayMode tests: hitting an enemy sets then clears the flash within
      a short window; killing an enemy spawns a burst that cleans itself
      up.

### Feature: Screen feedback (optional polish, Game + Gameplay)

- [ ] `Assets/Scripts/Game/Shake.cs` (pure): given an intensity and time,
      returns a decaying camera offset; EditMode-tested to decay to zero.
- [ ] A `CameraShake` MonoBehaviour applies it to `Main Camera` on player
      hit / enemy death. Optional — cutting it does not fail the
      milestone.

## UI Features

### UI Feature: Sprite art pass (Artist)

- [ ] PNGs under `Assets/Sprites/` for a player ship/character, an enemy,
      a bullet, and a tiling background — a neon-on-dark arcade look that
      matches the dark camera background.
- [ ] Each imports as a 2D `Sprite` with consistent pixels-per-unit for
      the arena scale.
- [ ] Sprites are assigned to the right renderers: `Player` → player
      sprite, `Enemy` prefab → enemy sprite, `Bullet` prefab → bullet
      sprite; the background renders behind all gameplay and does not
      occlude it.
- [ ] No placeholder squares remain on the player, enemy, or bullet.

### UI Feature: Main menu & flow (UI)

- [ ] `Assets/Scenes/MainMenu.unity` with a title and Play / Quit
      controls (uGUI buttons or key prompts). Play loads `Main.unity`;
      Quit exits play/app.
- [ ] Build Settings contain both scenes with `MainMenu` at index 0 and
      `Main` at index 1.
- [ ] The Game Over panel gains a "Main Menu" option that returns to
      `MainMenu`.
- [ ] A test asserts both scenes are in Build Settings in the correct
      order and that the Play control loads the gameplay scene.

## Architecture Notes

Art via the Artist/ComfyUI backend (`[agents.artist]` in `swizz.toml`,
ComfyUI workflow under `.swizz/workflows/`). VFX via `ParticleSystem` +
`SpriteRenderer` tint; the death burst is a pooled or self-destroying
prefab. Menu transitions via `SceneManager.LoadScene`. Camera shake stays
pure-logic for testability.

## Out of Scope

Multiple enemy types / bosses, power-ups & upgrades, persistence /
high-score tables, and mobile/touch input — future milestones.

## Dependencies

M1 + M2 (a complete, playable round to dress up).
