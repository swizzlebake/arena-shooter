# Milestone 1 — Core Loop: Move, Shoot, Kill

The playable nucleus of a top-down arena survival shooter, built in
Unity 6 / C# 2D. The player moves and aims with the Input System, fires
bullets that destroy enemies on contact, and a basic enemy seeks the
player and damages it on touch. Everything is placeholder colored
squares — waves, score, HUD, audio, and real art come in later
milestones. The win for this milestone: on **Play**, you can move around
the arena, shoot enemies that chase you, and die when they reach you.

## Context

The scene skeleton already exists and must be **built on, not rebuilt**:
`Assets/Scenes/Main.unity` is in Build Settings (index 0) and contains
`Main Camera` (orthographic, size 6, at origin), plus empty anchors
`Player` (tag `Player`), `EnemySpawner`, and `GameManager`. The arena is
centred on the world origin; bounds are roughly x ∈ [-9, 9],
y ∈ [-5, 5]. See `.swizz/context.md` for the assembly layout and the
hard constraints (trigger-based combat only — **no bounce physics**;
kinematic `Rigidbody2D.MovePosition` movement; new Input System only).

## Features

### Feature: Core types & math (Game assembly, MonoBehaviour-free)

- [ ] `Assets/Scripts/Game/Game.asmdef` is a plain assembly with **no
      `MonoBehaviour`** types (UnityEngine `Vector2` for math is fine).
- [ ] `Assets/Scripts/Game/Arena.cs` holds arena half-extents and
      `Vector2 Clamp(Vector2 p)` that keeps a point inside the bounds;
      default half-extents (9, 5).
- [ ] `Assets/Scripts/Game/HealthPool.cs` is pure: `Max`, `Current`,
      `ApplyDamage(int)`, `IsDead`; `Current` never drops below 0.
- [ ] `Assets/Scripts/Game/AimMath.cs` exposes `Direction(from, to)`
      returning a unit vector that is zero-safe (returns a defined
      fallback when from == to).
- [ ] EditMode tests under `Assets/Tests/EditMode/` cover: in-bounds
      points are unchanged and out-of-bounds points snap to the edge;
      `ApplyDamage` reduces, clamps at 0, and flips `IsDead`; `Direction`
      is unit length and zero-safe.

### Feature: Player movement & aiming (Gameplay)

- [ ] `Assets/Scripts/Gameplay/Gameplay.asmdef` references `Game`.
- [ ] `Assets/Scripts/Gameplay/PlayerController.cs` is a `MonoBehaviour`
      on the existing `Player` object: reads a move vector from the Input
      System and moves via `Rigidbody2D.MovePosition` at a serialized
      speed; every resulting position passes through `Arena.Clamp`.
- [ ] The player aims toward the mouse pointer / right stick and exposes
      the current aim direction for the weapon to read.
- [ ] Input is routed via the Input System (`InputActionAsset` /
      `PlayerInput`), never legacy `UnityEngine.Input`.
- [ ] The `Player` object is given a `SpriteRenderer` (placeholder solid
      square), a `Rigidbody2D` (gravityScale 0), and a trigger
      `Collider2D`.
- [ ] A PlayMode test drives a move input for several frames and asserts
      the player translates and never leaves the arena bounds.

### Feature: Shooting (Gameplay)

- [ ] `Assets/Scripts/Gameplay/Weapon.cs` fires on the Input System
      "fire" action at a serialized fire-rate, spawning a bullet at the
      muzzle along the player's current aim direction.
- [ ] `Assets/Scripts/Gameplay/Bullet.cs` + `Assets/Prefabs/Bullet.prefab`:
      `Rigidbody2D` (gravityScale 0) + trigger collider; moves at a
      serialized speed; despawns after a serialized lifetime or on
      leaving the arena.
- [ ] A PlayMode test fires the weapon and asserts a bullet is spawned
      and travels along the aim direction.

### Feature: Enemy & trigger combat (Gameplay)

- [ ] `Assets/Scripts/Gameplay/Enemy.cs` + `Assets/Prefabs/Enemy.prefab`:
      seeks the player each frame via `Rigidbody2D.MovePosition` at a
      serialized speed; owns a `HealthPool`; trigger collider; tag
      `Enemy`.
- [ ] A bullet contacting an enemy (resolved in `OnTriggerEnter2D` via
      `GetComponent`, **not** by physics bounce) applies damage and
      despawns the bullet; the enemy despawns when `HealthPool.IsDead`.
- [ ] The player owns a `HealthPool`; an enemy touching the player
      applies contact damage on a cooldown, and the player reports death
      at 0 HP.
- [ ] PlayMode tests: a bullet overlapping an enemy destroys the enemy
      and consumes the bullet; an enemy overlapping the player reduces
      player health.

### Feature: Playable scene wiring

- [ ] `Main.unity` keeps its `Main Camera`. The `Player` anchor is fully
      configured (renderer + body + collider + `PlayerController` +
      `Weapon`) and the input actions are assigned.
- [ ] A minimal placeholder spawn makes the scene playable on Play — e.g.
      a temporary component on `EnemySpawner` instantiates 2–3 enemies at
      the arena edges at start. (Real wave logic is M2; a hard-coded
      burst is fine here.)
- [ ] Collision setup ensures bullets hit enemies and enemies hit the
      player, while the player's own bullets do **not** hit the player.
- [ ] A PlayMode smoke test enters Play, waits ~2 seconds, and asserts:
      the player exists, at least one enemy exists, and no exceptions
      were logged.

### Feature: Assembly & test structure

- [ ] `Game.asmdef`, `Gameplay.asmdef` (refs `Game`), and both test
      asmdefs compile.
- [ ] `Assets/Tests/EditMode/EditMode.asmdef` references **only** `Game`;
      `Assets/Tests/PlayMode/PlayMode.asmdef` references `Game` +
      `Gameplay`.
- [ ] Both test suites are discoverable and pass in the Editor test
      runner.

## Architecture Notes

Kinematic movement and trigger collisions only — no `Rigidbody2D`
forces, no restitution/bounce, no "feel" constants needing a human
play-test. Placeholder art is a `SpriteRenderer` with a built-in square
sprite plus a tint. Death simply despawns for now; the game-over screen
and restart arrive in M2.

## Out of Scope

Waves/difficulty curve, score, HUD, audio, real sprite art, VFX, main
menu, restart — all M2/M3.

## Dependencies

The committed scene skeleton, Input System, and Test Framework are
already in the project.
